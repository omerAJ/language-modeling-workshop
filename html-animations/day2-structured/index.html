Three concrete steps

  • Define a fixed vocabulary of discrete symbols.
  • Split input text into pieces from that vocabulary.

Tokenization
Word-Level

Approach 1: one word = one token — simple, but brittle

Running sentence under word-level tokenization

If all words are in the vocabulary:
[researchers] [fine-tune] [large-scale] [models]
Hyphenated compounds like fine-tune and large-scale are typically absent from fixed word vocabularies. They collapse to unknown tokens:
[researchers] [UNK] [UNK] [models]

What works

  • One word = one token. Short sequences for common text.
  • Well-studied words carry full semantic weight as a single unit.

Why it breaks

  • Language is open-vocabulary. English Wikipedia alone has ~1.5M unique word forms.
  • New terminology — "GPT-4", "LoRA", "RLHF" — constantly creates unknowns.
  • No reuse of shared pieces: "run", "running", "runner" are all separate vocabulary entries.
! Word-level is efficient for familiar words, but brittle. Any new terminology, hyphenated compound, or non-English text becomes an unknown token.
Tokenization
Character-Level

Approach 2: one character = one token — flexible, but expensive

Running sentence under character-level tokenization

[r][e][s][e][a][r][c][h][e][r][s] [ ] [f][i][n][e][-][t][u][n][e] [ ] [l][a][r][g][e][-][s][c][a][l][e] [ ] [m][o][d][e][l][s]
Word-level (best case)
4 tokens
Character-level (same sentence)
~40 tokens
Scaling: 500-token passage
~2,500 character tokens

What works

  • No unknown tokens — ever.
  • Handles new words, typos, code, any language.

Why it is not the default

  • Sequences become \(5\text{–}10\times\) longer.
  • Transformer self-attention is \(O(n^2)\) in sequence length, so \(5\times\) longer means roughly \(25\times\) more compute.
  • The model spends capacity reassembling words from letters on every forward pass.
! Character-level coverage is perfect, but the compute cost is prohibitive. Quadratic attention scaling makes this impractical at real sequence lengths.

Subword Strategy

Approach 3: subword tokens — the practical default

Running sentence under subword tokenization

[research] [ers] [fine] [-] [tune] [large] [-] [scale] [model] [s]

Frequent stems become single tokens. Hyphens, prefixes, and suffixes are their own reusable pieces — no word is ever fully unknown.

Word-level

4 tokens

Short, but brittle. Breaks on hyphens, new jargon, multilingual text.

Character-level

~40 tokens

Thorough, but very long. Quadratic attention cost makes it impractical at scale.

Subword

~10 tokens

Near-word efficiency with no unknown tokens. GPT-4 averages ~0.75 words per token for English prose.

+ Subword gives near-word efficiency for common text and full coverage for rare strings. This is why it became the universal default in every modern LLM.
i There is one tradeoff: characters that end up inside the same token chunk are no longer individually visible to the model. We will see exactly when this matters at the end of this section.
Tokenization
BPE

How does the tokenizer learn these splits?

Subword splits don't come from rules or dictionaries. They are learned from data using Byte Pair Encoding (BPE).

Start: characters
Count adjacent pairs
Merge most frequent
Repeat N times

The core rule

BPE starts with every word split into individual characters. It repeatedly finds the most frequent adjacent pair in the corpus and merges it into a single new token. After N merges, you have a vocabulary of N reusable subword pieces.

Why it works

BPE is driven by frequency, not grammar. It discovers meaningful-looking pieces — stems, prefixes, suffixes — not because it understands linguistics, but because those strings appear together often. Compression and language structure happen to align.

i BPE is a frequency-based compression algorithm. It builds the vocabulary the training data demands, not the one a linguist would design.
? To see exactly how this works, let's run BPE on a corpus you'll recognise.

Tokenization
Why It Matters

Why tokenization shapes what the model can and cannot reason about

The mechanism

The model's computations run on token vectors — one vector per token piece. There is no separate vector for an individual character. Character-level reasoning requires the model to reconstruct structure from subword representations — something it is not architecturally designed to do.

"strawberry" → [straw] [berry] 2 token vectors. No individual letter vectors.

Cosma et al., "The Strawberry Problem: Emergence of Character-level Understanding in Tokenized Language Models", EMNLP 2025.

Tasks where this creates difficulty

  • Counting specific letters in a word.
  • Extracting the nth character from a string.
  • Detecting repeated characters.
  • Reversing a string exactly.

Important nuance

Tokenization is a structural contributor, not the only cause. Cosma et al. showed that character-level capabilities can emerge late in training, and a lightweight architectural modification significantly improves them. Tokenization creates a structural disadvantage, not an absolute ceiling.

! The model is fluent in token pieces. It is only indirectly aware of individual letters. Keep this in mind when designing prompts, tasks, and evaluations.
+ Tokenization is the first design decision in any language model. It defines the vocabulary, shapes the sequence length, and sets the perceptual granularity of every computation that follows.

Embeddings
Key Properties

Embeddings establish the layer-0 input state

What this establishes

  • A learned table \(E \in \mathbb{R}^{|V| \times d_{\text{model}}}\), where each token type has one trainable row.
  • Often one of the largest parameter blocks, scaling directly with \(|V| \times d_{\text{model}}\).
  • Lookup outputs \(X^{(0)}\) already match model width, so later blocks can consume them directly.
  • Geometry is distributed: useful structure appears as directions, neighborhoods, and relative offsets.

What still needs later layers

  • Sense is unresolved at input: one token row can support multiple intended meanings.
  • No token-token interaction has happened yet, so rows cannot use neighboring words.
  • Row identity alone does not specify sentence role, dependency structure, or discourse function.
  • Later layers must refine these initial vectors into context-dependent states.

Each row of the residual stream may have a very different scale. LN normalizes every row independently before the sublayer sees it.

Token-to-token mixing. Each token attends to all earlier tokens and gathers relevant context.
Residual Add (+)

Row-wise transformation. The same two-layer MLP applied independently to every token’s updated row.

Greedy
Always pick the top token. Deterministic, but tends to produce repetitive outputs for long sequences.
\(x_{t+1} = \arg\max_v\, p_v\)
Temperature \(T\)
Divide logits by \(T\) before softmax. \(T \to 0\) approaches greedy; \(T > 1\) flattens the distribution for more varied outputs.
\(p_v \propto \exp(\ell_v / T)\)
Top-\(p\) (Nucleus)
Keep only the smallest set of tokens whose cumulative probability reaches \(p\). Sample from that set — adapts to each distribution.
sample from \(\{v : \sum p_{v'} \ge p\}\)
